# 🛑 Hold On MCP - 让 AI 停下来听你说话

## 痛点

用 AI 编程助手时，你是不是经常遇到：

- **AI 废话连篇**：问个简单问题，它给你背诵一遍项目背景
- **理解偏差**：讨论到一半，它突然往错误方向长篇大论
- **点数浪费**：每次错误输出都在消耗你的额度

**问题根源**：AI 一旦开始输出，你只能等它说完，无法中途干预。

## 解决方案

Hold On MCP 是一个 MCP 工具，通过**浏览器反馈页面**强制拦截 AI 的执行流程。

AI 完成每次实质性输出后，会自动弹出反馈页面：
- **留空点确认** → 满意，继续
- **输入调整指令** → AI 立即修改，不重新开启新对话
- **粘贴截图** → 直接给 AI 看问题

## 安装（零配置）

在 MCP 配置文件中添加：

```json
{
  "mcpServers": {
    "hold-on": {
      "command": "npx",
      "args": ["-y", "hold-on-mcp"],
      "env": {
        "HOLD_ON_THEME": "auto"
      }
    }
  }
}
```

主题选项：`auto`（跟随系统）/ `light` / `dark`

## 核心秘籍：紧箍咒

在 `.windsurfrules` 或 Cascade 全局规则中添加：

```
**强制交互协议：**
1. 你现在拥有 `request_approval` 工具。
2. **在完成每次实质性输出后**（如回答问题、写代码、改文件），必须调用 `request_approval` 简述你做了什么，等待用户确认。
3. 如果用户通过弹窗给了你调整指令，你必须在**当前这一轮对话**中立即修改，不准重新开启新对话。
4. 用户留空点确认 = 满意；用户输入内容 = 需要调整。
```

## 特性

- ✅ 浏览器反馈页面（支持图片粘贴）
- ✅ 暗黑模式支持
- ✅ 5 分钟超时自动确认
- ✅ 零安装（npx 自动下载最新版本）
- ✅ 跨平台（macOS / Windows）

## 为什么有效？

- **不是"查字典"，是"设路障"** - AI 必须停下来等你
- **阻塞效果** - 页面不关闭，AI 就没法往下走
- **当前对话修改** - 不浪费上下文，不重新开启新对话
